\section{Preliminaries}
\label{Pre}
\subsection{Quantum Bits and Quantum Gates}
\label{Chap:Pre-qubits}
Quantum computers internally represent data as \emph{qubits}, which are quantum systems that can
take on the quantum states $\ket{0}$ and $\ket{1}$, each known as \emph{basis states}.

%% Additionally, qubits can exist in quantum states that are a linear combination of $\ket{0}$ and $\ket{1}$,
%% which is called \emph{superposition}:
%% \begin{equation}
%% \ket{\psi} = \alpha_0 \ket{0} + \alpha_1 \ket{1}, \quad \alpha_0, \alpha_1 \in \mathbb{C}
%% \end{equation}

%% Because these coefficients $\alpha_0$ and $\alpha_1$ are complex numbers, they can also be expressed as a complex exponential $a\cdot e^{i \theta}$, where $\theta$ is often referred to as a \emph{phase}.

Qubits can be taken together as tensor products to create multiqubit states.
\begin{equation}
\label{eq-multi-qb}
\ket{\psi} = \ket{\psi}_0 \otimes \ket{\psi}_1 =(\alpha_{00} \ket{0}_0 + \alpha_{01} \ket{1}_0) \otimes
(\alpha_{10} \ket{0}_1 + \alpha_{11} \ket{1}_1)
\end{equation}
The tensor product of several basis states is known as a \emph{computational basis state},
and is written as a bit string consisting of the values in the component tensor multiplicands.
\begin{equation}
\ket{x_0}\otimes\ket{x_1}\otimes\cdots\otimes\ket{x_{n-1}}=\ket{x_0 x_1 \cdots x_{n-1}}=\ket{\mathbf{x}},
\mathbf{x} \in \{0,1\}^n
\end{equation}

%% These computational basis states can also be in linear superposition.
%% \begin{equation}
%% \ket{\psi} = \sum_{\mathbf{k} \in \{0,1\}^n} \alpha_{\mathbf{k}}\ket{\mathbf{k}},
%% \sum_{\mathbf{k} \in \{0,1\}^n} \alpha_{\mathbf{k}}^2 = 1
%% \end{equation}

{\it Quantum gates} describe qubit mappings. The main ones concerning this work
are demonstrated in Fig.~\ref{fig-gates}. The $T/T^{\dagger}$, $S/S^{\dagger}$, and $Z$
gates will be referred to as {\it phase gates}.

All of the gates in ~\figref{fig-gates} except $T/T^{\dagger}$ and Toffoli are known as
{\it Clifford gates}. For fault-tolerant quantum computers, Clifford gates are relatively
inexpensive to implement. However, without $T/T^{\dagger}$ gates, computations that display
quantum advantage cannot be implemented, which have a much bigger implementation cost. Thus,
{\it T-count}, the number of $T/T^{\dagger}$ gates, and {\it T-depth}, the longest such
serial chain of them in a circuit, are often used as a cost metric~\cite{amy-meet-in-middle}.
The set of gates in Fig.~\ref{fig-gates} is known as the {\it Clifford+T} gates, which are
universal for fault-tolerant quantum computing. The Toffoli gate can be implemented as a
combination of the other gates.

In addition, versions of the CNOT and Controlled-Z gate exist which have multiple controls,
which means the phase is applied when all the control bits are 1, just like the CNOT gate.
These are not native functions to a quantum computer, and will need to be constructed from
the other gates. The rest of this paper goes into various ways to construct them.

All of the gates in Fig.~\ref{fig-gates} can be composed into {\it quantum circuits}.
A subclass of quantum circuits, which applies an X-gate on one output qubit
if a Boolean function of the input states is true or false, respectively, is known as a
{\it quantum oracle}. We cover a variation of this type of circuit in the next section.

\subsection{Phase Oracles}
\label{Pre:oracle}
A {\it phase oracle} is one such quantum circuit that implements a Boolean function as a
quantum circuit, implementing a mapping of a computational basis state
$\ket{\mathbf{x}} \mapsto e^{i \pi f(\mathbf{x})}\ket{\mathbf{x}}$, where $f(\mathbf{x})$
is a Boolean function $\{0,1\}^n \rightarrow \{0,1\}$. Phase oracles are key elements in
algorithms such as Grover's Algorithm.

A phase oracle can be constructed from a quantum oracle as in~\figref{fig-phase-n4}. The
$F$-controlled NOT gate is a quantum circuit that inverts the target qubit, when the Boolean
function $F(x_1,x_2,\cdots,x_n)$ is true. This gate is depicted in~\figref{fig-urtof}, and is
sometimes known as a {\it single target gate}. A Toffoli gate is an example of such a single
target gate, where $F$ is the Boolean AND of the two inputs.Placing H-gates on either side of
the target bit, as depicted in~\figref{fig-phase-n4} utilizes the identity $HXH = Z$, where
$H$, $X$, and $Z$ are the transformations effected by the $H$, $X$, and $Z$ gates respectively.
This means this circuit applies a phase of $-1$ when $F$ is true.

\subsection{Synthesizing Phase Oracles from ESOP}
\label{Pre:OracleEsop}
A transformation $\ket{x}\ket{y} \rightarrow (-1)^{x \oplus y}\ket{x}\ket{y}$ can be
realized using the below circuit
\begin{figure}[h]
  \centering
  \scalebox{1.0} {
    \Qcircuit @C=0.5em @R=0.2em @!R {
      \lstick{\ket{x}} & \qw & \gate{Z} & \qw  \\
      \lstick{\ket{y}} & \qw & \gate{Z} & \qw
    }
  }
\end{figure}

Thus, given an Exclusive Sum of Products expression of a Boolean function
$f(\mathbf{x}) = x_a x_b \cdots x_g \oplus x_i x_j \cdots x_m \oplus \cdots$, we
can implement its phase oracle by inserting controlled phase gates that are activated
by the {\it cubes}, or the product terms of the ESOP, as Boolean functions. Each of these
cubes is composed of {\it literals} consisted of positive or negative values of an input variable.

This method is used by Qiskit's Phase Oracle \cite{bib-phaseoracle}. Our proposal involves
the effective use of such an Exclusive Sum of Products to implement such a phase oracle.

\begin{example}
  \label{ex-esop-synth}
  We synthesize the Boolean function
  $\bar{x_a}\bar{x_c} \oplus \bar{x_a} x_b \bar{x_c}\bar{x_d} \oplus x_a x_b \oplus x_a \bar{x_c} x_d$
  into a quantum circuit. The first term $x_a x_b$ is merely a two input Controlled-Z gate so we insert
  such a gate between the qubits containing $x_a$ and $x_b$. $\bar{x_a}\bar{x_c}$, $x_a \bar{x_c} x_d$, and
  $\bar{x_a}\bar{x_b}\bar{x_c}\bar{x_d}$ have similar Controlled-Z gates, so we generate those.
  ~\figref{fig-esop-synth} shows the result of the synthesis, where the open control bits signify negated
  inputs, and multiple controls mean that the gate activates as an AND of the controls.
\end{example}

\subsection{CNOT+T Circuits and Phase Polynomials}
\label{Pre:CnotPhase}

Previous research has demonstrated that some phase oracles can be implemented using only
CNOT and phase gates (CNOT+T)~\cite{bib-amy-cnot}. When done in such a manner,
the cumulative action of the phase gates $f(\mathbf{x})$ in the mapping
$\ket{\mathbf{x}} \mapsto e^{i \pi f(\mathbf{x})}\ket{\mathbf{x}}$
as defined in Eq.~\ref{eq-boolean-fourier}, known as its{\it phase polynomial}.

\begin{equation}
  \begin{aligned}
    \label{eq-boolean-fourier}
    f(\mathbf{x}) = \sum_{\mathbf{k} \neq 0} \hat{f}(\mathbf{k}) \cdot \chi_k(\mathbf{x}),
    \hat{f}(\mathbf{k}) \in \mathbb{R}
  \end{aligned}
\end{equation}

Where $\chi_k(\mathbf{x}) = ( k_0 x_0 \oplus k_1 x_1 \oplus \cdots \oplus k_{n-1} x_{n-1})$
for $\mathbf{k} \in \{0,1\}^{n}$. 

This can be read directly from the circuit using the following method.

\begin{itemize}
\item Initialize an empty phase polynomial with value 0.
\item Take each phase gate as described in ~\figref{fig-gates}, take the power
  of $e^{i\pi}$ that it drives as a coefficient, and multiply it by the function
  that drives it
\item Add the phase above to the phase polynomial.
\item Repeat until all phase gates have been processed.
\end{itemize}

The resulting expression from this method is sometimes called a circuit's
\emph{sum of paths}~\cite{bib-amy-cnot}.

\begin{figure}[t]
  \centering
  \scalebox{0.7} {
    \input{img/fig-toff-mark-matroid2}
  }
  \caption{CNOT+T Circuit}
  \label{fig-toff-mark-matroid2}
\end{figure}

\begin{example}
  Let's calculate the phase polynomial for~\figref{fig-toff-mark-matroid2}. The topmost
  T-gate is driven directly by the input, which is initialized to $x_a$. This gives us
  $\frac{1}{4} x_a$. The phase polynomial is now $f(\mathbf{x}) = \frac{1}{4} x_a + \cdots$.
  The T-gate below that is driven by the output of a CNOT gate with control from the qubit
  initialized to $x_c$ and target to $x_b$. This gives us $\frac{1}{4} x_b \oplus x_c$. The
  phase polynomial so far now is
  $(\mathbf{x}) = \frac{1}{4} x_a + \frac{1}{4} x_b \oplus x_c + \cdots$. The rest of the
  calculation can be completed to show that the phase polynomial is:

  \begin{equation}
    \label{eq-pp-ex}
    \begin{aligned}
      &f(\mathbf{x}) = \frac{1}{4}x_a + \frac{1}{4}x_b + \frac{1}{4}x_c - \frac{1}{4}(x_a \oplus x_b) \\
      &\qquad -\frac{1}{4}(x_a \oplus x_c) - \frac{1}{4}(x_b \oplus x_c) + \frac{1}{4}(x_a \oplus x_b \oplus x_c)\nonumber
    \end{aligned}
  \end{equation}

\end{example}

Conversely, CNOT+T circuits can be synthesized from phase polynomials.
Eq.~\ref{eq-toff-bool} demonstrates another such phase polynomial. It should be
straightforward to check that this phase polynomial computes $x_a \cdot x_b \cdot x_c$ by
substituting values for $x_a, x_b, x_c$. That means it can implement the three input
Controlled-Z gate.

\begin{equation}
  \label{eq-toff-bool}
  \begin{aligned}
    &x_a \cdot x_b \cdot x_c = \frac{1}{4}x_a + \frac{1}{4}x_b + \frac{1}{4}x_c - \frac{1}{4}(x_a \oplus x_b) \\
    &\qquad -\frac{1}{4}(x_a \oplus x_c) - \frac{1}{4}(x_b \oplus x_c) + \frac{1}{4}(x_a \oplus x_b \oplus x_c)
  \end{aligned}
\end{equation}

Corresponding $\chi_k(\mathbf{x})$ functions in the phase polynomial can be implemented as a
network of CNOT and X gates. Multiple methods have been developed to generate this
network for CNOT circuits~\cite{bib-barenco-elementary,bib-amy-cnot}, and the appropriate
one can be selected depending on the optimization goals. The coefficients associated with
the $\chi_k(\mathbf{x})$ functions in the phase polynomial describe the phase that needs to
be driven as multiples of $i\pi$ by using phase gates such as $S$ and $T$ (i.e. $3/4$ will
be driven as an $S$-gate followed by $T$-gate). Those phase gates will be placed at the
part of the CNOT and X network that calculates the state that drives it. The next example
elaborates on how Fig.~\ref{fig-toff-mark-matroid} implements Eq.~\ref{eq-toff-bool}.

\begin{example}
  The first term in Eq.~\ref{eq-toff-bool} is the function $x_a$ with the coefficient
  $\frac{1}{4}$. To drive this phase, a T-gate is placed on the input $x_a$. Next, we try to
  drive the term for $x_b \oplus x_c$, which has a coefficient of $-\frac{1}{4}$. To do
  this, we place a CNOT gate with $x_c$ as the control and $x_b$ as the target. We then
  place a $T^{\dagger}$ gate after the target. The rest of the circuit can be similarly
  constructed. Many ways of sequencing the terms in Eq.~\ref{eq-toff-bool} exist, such as
  the matroid partitioning algorithm in~\cite{bib-amy-matroid}, but the details will not be
  important to our proposal.
\end{example}

  \centering
  \scalebox{0.7} {
    \input{img/fig-toff-mark-matroid}
  }
  \caption{Implementing $x_a \cdot x_b \cdot x_c$ using CNOT and T gates}
  \label{fig-toff-mark-matroid}
\end{figure}

\subsection{Boolean Fourier Transform}
\label{Pre:Four}
Observe that the $\chi_k(\mathbf{x})$ from Eq.~\ref{eq-boolean-fourier} form an orthonormal basis with respect
to the below inner product.

\begin{equation}
  \label{eq-inner-prod}
  \langle f(\mathbf{x}) , g(\mathbf{x}) \rangle = \frac{1}{2^n} \sum_{\mathbf{x}} (-1)^{f(\mathbf{x})} (-1)^{g(\mathbf{x})}, \quad \mathbf{x} \in \{0,1\}^n
\end{equation}

for $n = |\mathbf{x}| \leq 3$.

\begin{figure*}[t]
  \centering
  \begin{minipage}{\textwidth}
    \centering
    \scalebox{0.9} {
      \input{img/fig-series-4inp-inv}

    \label{fig-series-4inp-inv}
  \end{minipage}
  \begin{minipage}{\textwidth}
    \centering
    \scalebox{0.9} {
      \input{img/fig-series-4inp}
    }
    \subcaption{Phase oracle implemented with gates from Boolean Fourier Transform}
    \label{fig-series-4inp}
  \end{minipage}
  \caption{Phase oracles for $x_a \bar{x_b} x_c \oplus x_b \bar{x_c} x_d$}
  \label{fig-series-all}
\end{figure*}
\begin{figure}[t]
  \centering
  \scalebox{1.0} {
    \input{img/fig-series-mct}
  }
  \caption{Phase oracle generated by ESOP expression}
  \label{fig-series-4inp-mct}
\end{figure}
That means, we can calculate $\hat{f}(\mathbf{k})$ using the below inner product

\begin{equation}
  \label{eq-fhat-prod}
  \hat{f}(\mathbf{k}) = \langle f(\mathbf{x}), \chi(\mathbf{x})_{\mathbf{k}} \rangle \forall \mathbf{k} 
\end{equation}

$\hat{f}(\mathbf{k})$ is also known as the Boolean Fourier transform of $f(\mathbf{x})$~\cite{bib-odonnell}
\footnote{All equations that are taken from~\cite{bib-odonnell} are written here
with coefficients in $\{0,1\}$ and $\chi_{\mathbf{k}}(\mathbf{x})$ functions from ~\cite{bib-amy-matroid}.}.
\begin{example}
As an example, let's derive the coefficients for Eq.~\ref{eq-toff-bool} with $\mathbf{k} = (k_{x_a}, k_{x_b}, k_y)$.
For $\mathbf{k} = 100$, $\chi_{100} = x_a$. Table~\ref{table-ex-100} enumerates the values and the term $(-1)^{f(\mathbf{x})} (-1)^{\chi_{100}(\mathbf{x})}$ for the inner product.
\begin{table}[h]
  \begin{center}
    \begin{tabular}{c|c|c|c|c|c}
      \hline
      $x_a$ & $x_b$ & $y$ & $f(\mathbf{x})$ & $\chi_{100}$ & $(-1)^{\chi_{100}(\mathbf{x})} (-1)^{f(\mathbf{x})}$ \\\hline
      0 & 0 & 0 & 0 & 0 & 1\\\hline
      0 & 0 & 1 & 0 & 0 & 1\\\hline
      0 & 1 & 0 & 0 & 0 & 1\\\hline
      0 & 1 & 1 & 0 & 0 & 1\\\hline
      1 & 0 & 0 & 0 & 1 & -1\\\hline
      1 & 0 & 1 & 0 & 1 & -1\\\hline
      1 & 1 & 0 & 0 & 1 & -1\\\hline      
      1 & 1 & 1 & 1 & 1 & 1\\\hline
    \end{tabular}
    \caption{The value of $\chi_{100}$ and the inner product $\langle f(\mathbf{x}), \chi_{100} \rangle$}
    \label{table-ex-100}
  \end{center}
\end{table}

Summing the rightmost column, as in Eq.~\ref{eq-inner-prod}:
\begin{align}
  &\hat{f}(100) &= \langle f(\mathbf{x}), \chi_{100} \rangle &= \frac{1}{8} ( 1 + 1 + 1 + 1 - 1 - 1 - 1 + 1 ) \\
  &\hat{f}(100) &= \frac{1}{4}\nonumber
\end{align}
Thus,
\begin{equation}
  f(\mathbf{x}) = \frac{1}{4}x_a + \sum_{\mathbf{k} \neq 000,100} \hat{f}(\mathbf{k}) \cdot \chi_{\mathbf{k}}
\end{equation}
The rest of the calculation is omitted for brevity, but readers can verify that it results in Eq.~\ref{eq-toff-bool}.
\end{example}

Synthesizing the phase polynomial of a Boolean function using the Boolean Fourier
transform for threevariables or fewer has an interesting consequence. First, the
number of odd multiples of $i\frac{\pi}{4}$ of $\hat{f}(\mathbf{k})$ corresponds to
the T-count needed to implement $f(\mathbf{x})$~\cite{bib-amy-rm}. This is easy to
check, since any even multiple of $i\frac{\pi}{4}$ can be implemented using only
S and Z gates, which do not take any T-cost, and any odd multiple can be realized
using S and Z gates, plus one T-gate. Since each $\hat{f}(\mathbf{k})$
is unique to each $f(\mathbf{x})$ when $|\mathbf{x}| \leq 3$,
$\hat{f}(\mathbf{k})$ has the optimal T-count needed to implement
$f(\mathbf{x})$ as a phase oracle when $|\mathbf{x}| \leq 3$.

\subsection{CNOT+T Circuit Optimization}
\label{Mot:CnotOpt}
CNOT+T circuits are notable because the T-gates in them can be freely rearranged to optimize T-depth.
Techniques such as matroid partitioning guarantee an optimal T-depth
for them~\cite{bib-amy-matroid}. Given enough ancilla, they can be rearranged to
arbitrarily small T-depth, down to a T-depth of 1 in some cases~\cite{bib-selinger-tdepth}.

Additionally, there is a straightforward way to reduce the T-count for such circuits.
Because the phase is a scalar, the calculated phase polynomial may combine the contributions of
several T-gates into a phase that can be driven by other phase gates such as Z and S, which
have lower implementation cost. This means that the resulting phase polynomial can sometimes be
implemented with fewer T-gates than the original circuit.

\begin{example}
  \label{ex-series}
  We calculate the phase polynomial for a phase oracle that calculates
  $x_a \bar{x_b} x_c \oplus x_b \bar{x_c} x_d$.~\figref{fig-series-all} displays realizations of this phase
  oracle.~\figref{fig-series-4inp-mct} shows the realization of the phase oracle as Controlled-Z gates
  from~\figref{Pre:OracleEsop}.~\figref{fig-series-4inp-inv} shows each of the Controlled-Z gates generated
  as X-gates and the CNOT+T implementation of the three input Controlled-Z in~\figref{fig-toff-mark-matroid}.
  ~\figref{fig-series-4inp} shows the implementation of Controlled-Z gates by using the Boolean Fourier
  Transform method from~\secref{Pre:Four}. It is easy that all of these are phase oracles for
  $x_a \bar{x_b} x_c \oplus x_b \bar{x_c} x_d$ by substituting values for $x_a,x_b,x_c,x_d$ and checking
  the resulting phase.

  Following the method described in \secref{Mot:CnotOpt}, we calculate the phase polynomial
  for~\figref{fig-series-4inp-inv} which gives us the following equation:

  \begin{equation}
    \label{eq-series1}
    \begin{aligned}
      &x_a \bar{x_b} x_c \oplus x_b \bar{x_c} x_d = \frac{1}{4}x_a + \frac{1}{4}\bar{x_b} 
      + \frac{1}{4}x_c - \frac{1}{4}(x_a \oplus \bar{x_b}) \\
      &- \frac{1}{4}(x_a \oplus x_c) - \frac{1}{4}(\bar{x_b} \oplus x_c)
      + \frac{1}{4}{(x_a \oplus \bar{x_b} \oplus x_c)} \\
      & +\frac{1}{4}x_b +\frac{1}{4}\bar{x_c} +\frac{1}{4}x_d 
        - \frac{1}{4}(\bar{x_b} \oplus x_c) - \frac{1}{4}(\bar{x_b} \oplus x_d) \\
      & - \frac{1}{4}(x_c \oplus x_d) + \frac{1}{4}(\bar{x_b} \oplus x_c \oplus x_d) 
    \end{aligned}
  \end{equation}

  
  This gives us a T-count of 14 and a T-depth of 6.

  Next, we do the same for~\figref{fig-series-4inp}.
  
  \begin{equation}
    \label{eq-series}
    \begin{aligned}
      &x_a \bar{x_b} x_c \oplus x_b \bar{x_c} x_d = \frac{1}{4}x_a - \frac{1}{4}x_b + \frac{1}{4}x_c
      + \frac{1}{4}(x_a \oplus x_b) \\
      & - \frac{1}{4}(x_a \oplus x_c) + \frac{1}{4}(x_b \oplus x_c)
      - \frac{1}{4}(x_a \oplus x_b \oplus x_c)  \\
      & + \frac{1}{4}x_b - \frac{1}{4}x_c + \frac{1}{4}x_d
      + \frac{1}{4}(x_b \oplus x_c) \\
      & + \frac{1}{4}(x_b \oplus x_d) - \frac{1}{4}(x_c \oplus x_d)
      - \frac{1}{4}(x_b \oplus x_c \oplus x_d)  
    \end{aligned}
  \end{equation}
  
  Here, we can combine terms with the same basis function and get the following equation:
  
  \begin{equation}
    \label{eq-series-simp}
    \begin{aligned}
      & x_a \bar{x_b} x_c \oplus x_b \bar{x_c} x_d = \frac{1}{4}x_a + \frac{1}{4}x_d
      - \frac{1}{4}(x_a \oplus x_b) \\
      & + \frac{1}{4}(x_a \oplus x_c) + \frac{1}{2}(x_b \oplus x_c)
      + \frac{1}{4}(x_b \oplus x_d) -\frac{1}{4}(x_c \oplus x_d)     \\
      & + \frac{1}{4}(x_a \oplus x_b \oplus x_c) - \frac{1}{4}(x_b \oplus x_c \oplus x_d) 
    \end{aligned}
  \end{equation}

  This means we can resynthesize this circuit with 6 fewer T-gates. Unlike~\figref{fig-series-4inp-inv},
  \figref{fig-series-4inp} has no negated inputs, so more of its terms can combine together. This shows
  the power of generating the Controlled-Z gates using this methodology.
  
\end{example}

\subsection{Boolean Logic Network}

A {\it Boolean logic network} or {\it logic network} is a simple Directed Acyclic Graph (DAG).
It has nodes, edges, primary outputs, and primary inputs. Each Boolean logic network represents
a Boolean function, and a \emph{node} represents a subfunction used to decompose that Boolean
function. It has zero or more {\it fanins}, or incoming edges, as well as zero or more
{\it fanouts} or outgoing edges. The {\it primary inputs (PIs)} are nodes without fanins in
the current network. The {\it primary outputs} are a subset of nodes in the network without
fanouts.

A Boolean function can be decomposed into such a Boolean logic network. A common choice is the
XOR AND Inverter Graph (XAG)~\cite{bib-meuli-xag}. This is a logic network where the nodes represent
XORs and ANDs, with any combination of their inputs or outputs negated. A naive way to construct such
an XAG graph is by using Pseudo Kronecker Reed Muller expansions~\cite{bib-pkrm,bib-exorcism}.
This initial logic network is called the function's \emph{subject graph}. The subject graph can then be
further refined by tools such as abc~\cite{bib-mishchenko-lut} to reduce the number of nodes, increasing
sharing.

Previous research~\cite{bib-meuli-xag,bib-meuli-ros,bib-meuli-mult} has demonstrated that the
number of nodes in a Boolean network is directly relevant to its T-count and T-depth, so this paradigm
is useful as a way to approach T-count/T-depth optimization by leveraging existing techniques to
optimize the number of nodes in a network. In particular, the {\it multiplicative complexity} of logic network
is the number of AND nodes in it, and its {\it multiplicative depth} is the longest chain of such AND nodes
from a primary input to a primary output~\cite{bib-meuli-xag}.

\subsection{Boolean Logic Network Based Quantum Circuit Synthesis}
\label{Subsec:lhrs}

Previous research~\cite{bib-lhrs-soeken,bib-meuli-xag} has detailed methods to use Boolean logic
networks to generate quantum circuits by breaking them down into components that can be more
easily mapped to quantum circuits. In this paradigm, each of the primary inputs is mapped to an
input qubit, and each of the primary outputs is mapped to an output qubit. The Boolean logic network
in each function is synthesized by using a quantum gate. Each node, which represents a Boolean
function $F$, is implemented as a single-target gate, such as that depicted in~\figref{fig-urtof}.

The action of a single-target gate on an ancilla is called {\it computation}, and the gate is
known as a {\it compute} operation. To preserve the measurement properties of the overall state,
there is a need to undo the actions of this operation, which is called {\it uncomputation}
\cite{bib-barenco-elementary}. The gates used to accomplish this are {\it uncompute} operations.
Therefore, the flow next produces uncompute logic. This produces a compute/uncompute pair.

The result of the method is shown in Fig.~\ref{fig-lut-qc}. To get from
Fig.~\ref{fig-lut-network} to Fig.~\ref{fig-lut-qc}, first, we take as input a Boolean
logic network synthesized from a Boolean expression using existing Boolean logic network
decomposition methods. Then this Boolean logic network is parsed by the algorithm to
create a quantum circuit of single-target gates. Any intermediate node is decomposed as two
single-target gates: one to compute, and the other to uncompute, acting on the same clean
ancilla. Any node that acts only on output qubits is given only one single-target gate.


\begin{example}
  \label{ex-lut}
  We start at node 1. Since this is an intermediate node, we place the single target gate
  $1$ to act on an ancilla, where $1$ applies the X operation to the ancilla when the Boolean expression
  in node $1$ is true. We then place its inverse $1^{\dagger}$ on the same ancilla to
  uncompute. Node 2 is similarly independent, so we do the same for node 2. Node 4 is
  dependent on both nodes 1 and 2, so we connect the target qubits from $1$ and $2$
  into $4$ as input, and place $4$, as well as its uncompute gate $4^{\dagger}$, on
  another ancilla. Finally, nodes 3 and 5 are output nodes, so we only generate their
  compute logic.

  Note in this diagram we denote qubits that a gate does not act on as dashed lines running
  across the gate, as opposed to qubits it acts on which terminate at the edges.
\end{example}

Further research in this field has explored how to reduce T-count and T-depth with such logic network
methods (\cite{bib-meuli-ros,bib-accel-lut}). However, most of the research concentrates
on reducing the T-count with quantum oracles. We demonstrate in the next section how these can be
further adapted to generate phase oracles while reducing T-count and T-depth.
